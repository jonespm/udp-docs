SIS Loading schema v. 1.1

Documentation


What is a loading schema?

It is a document that defines the data and formatting requirements necessary to load data into the Unizin Data Platform. In addition, the documentation below guides decisions you must make about aligning SIS data at your Institution to the Unizin Common Data Model.

During your UDP implementation, Unizin will provide guidance about the meaning of the data below and help you identify how to align your SIS data to the Unizin Common Data model.

How much data is required for each entity?

Person. The intent is to capture all individuals who have any relationship to teaching and learning practices, and who generate data in any digital tool used in teaching and learning. This includes advisors, tutors, students, instructors, instructional designers, faculty, teaching assistants, and other actors whose behaviors generate data in a teaching & learning environment or about whom data is generated in a teaching and learning environment.

Academic Term, Session, Campus, Academic organization, Course, Course Section, Section enrollment, and Course grade. It is expected that all records for these entities in your SIS are relevant to the UDP. These entities provide essential contextual information that describe your teaching and learning environments and individuals’ membership in those environments. Enrollments include not only student, but also instructors (with teaching assignments), TA, etc.

What are my next steps?

The first step in producing data to that model is correlating data in the SIS loading schema to data in your SIS. While some data is easy to correlate (e.g., a Person's first, middle, and last name), other data may not correspond exactly how data is modeled in your SIS. In these latter cases, please open a dialog with Unizin to determine how to proceed.

Second, many data elements in the Unizin Data Common Model have are limited to a finite set of predefined values, called "Option sets." When a particular element in the loading schema uses an Option set, you will need to translate your SIS data values to UCDM data values. For example, email addresses in UCDM have a EmailType data element whose values can be Home, Work, Organizational, and Other. If your SIS models an email's "type," its possible values may or may not perfectly overlap with the values in the UCDM Option set for EmailType. When you produce the SIS data for ingestion in the UDP, you will need to align the values of data elements that use an option set to the appropriate code in that Option set.

New entities and new or updated elements as part of the 1.1 version of the schema are marked with an asterisk (*). All entities and elements are expected by the system; this is intended only to clarify changes for institutions upgrading from 1.0.

Data type formats

When data of type date is requested, the format is YYYY-MM-DD. When data of type time is requested, the format is hh14:mm:ss. If the data does not exist for a particular record, or the mapping between your SIS and the UCDM is insufficient, leave the value blank.

File formats

Your data files must be UTF-8 encoded, comma-separated value (CSV) files with column headers. The header strings are described in the SIS Loading Schema document.

  • Produce one data file for each UCDM Entity below
  • Data files are full dumps (not deltas)
  • Data files are generated and pushed daily to the UDP
  • The field delimiter is a comma (,)
  • The value quoting character is a double quotes (")
  • The quote escape character is a backslash (\)
  • Non-applicable or otherwise unmapped fields should be left fully blank ("a",,"c")
  • Option set values are case-sensitive

Person

An individual participating, in any capacity, in the design, delivery, or evaluation of teaching and learning processes and environments.

The Person file should be created in a comma-separated (CSV) format with the naming convention person_<date>.csv , where is the date on which the file was generated, and the following headers:

Institutional affiliation

The association of a Person to an academic institution, and the description of the general or primary capacity in which they act at that institution.

The Institutional affiliation file should be created in a comma-separated (CSV) format with the naming convention institutional_affiliation_<date>.csv , where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Person ID PersonId The unique, SIS-generated internal Person ID for the person affiliated in an institution of higher education.
Directory information block DirectoryBlock

A variable used to describe Persons who are currently electing to block their directory information as per FERPA, or those who previously had a block but are requesting it be removed. A `null` value indicates that the related person does not have a FERPA directory block, and values may be shared in accordance with standard data agreements.   - This block is only valid for institutional affiliated records.

Option set: RefDirectoryInformationBlock

Campus*

The collection of departments, colleges, schools, and buildings that belong to a particular institution. All Universities have at least one campus. Some Universities are systems with multiple Campuses. These may include "virtual campuses" that represent online programs, which is common to many Unizin Members.

The Campus file should be created in a comma-separated (CSV) format with the naming convetion campus_<date>.csv where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Campus ID SisIntId The internal primary key used by the SIS to define a Campus record.
SIS External Campus ID SisExtId The unique, global ID for a Campus generated in yor institutional SIS for use by external tools such as Canvas, LTI applications, etc.
Name Name The full name of the Campus in the Institution's system (e.g., "Fort Wayne").
Abbreviation Abbreviation The abbreviation for the campus name. The intent is to capture the shorthand used in schedule of classes and other administravia to refer to the campus (e.g., "BL" for Bloomington).

Academic organization*

An Academic organization is a School, Program, College, or Department responsible for delivering instruction. A particular Academic organization is always unique to a campus. However, Academic organizations across different campuses may share the same Organization Code. For example, there may exist a Department of Sociology at multiple campuses of a University system — each instance can be uniquely identified, but all may share the same academic organization code.

The Academic organization file should be created in a comma-separated (CSV) format with the naming convention academic_organization_<date>.csv, where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Academic organization ID SisIntId The internal primary key used by the SIS to define an academic organization record.
SIS External Academic organization ID SisExtId The unique, global ID for an Academic organization generated in your institutional SIS for use by external tools such as Canvas, LTI applications, etc.
SIS Internal Campus ID CampusId The foreign key to the Campus to which the Academic organization belongs.
SIS Internal Parent Academic organization ID ParentId (optional) The foreign key to the parent Academic organization of this Academic organization.
Academic organization code Code A code for the Academic organization. The intended value is the shorthand used in schedule of classes and other administravia to refer to the Academic organization (e.g., "CAS").
Name Name The human-readable name of the Academic organization (e.g., "The College of Arts and Sciences").

Session*

An administrative unit that defines an actual instructional period during a term. Every Academic term is comprised of one or more Sessions. A Session is always unique to an Academic term and will have many Course offerings.

The Session file should be created in a comma-separated (CSV) format with the naming convetion session_<date>.csv, where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Session ID SisIntId The internal primary key used by the SIS to define a Session record.
Sis External Session ID SisExtId The unique, global ID for a Session generated in your institutional SIS for use by external tools such as Canvas, LTI applications, etc.
SIS Internal Term ID TermId The foreign key to the Academic term to which the Session belongs.
Name Name The name of the Session. The value is typically intended to be human-readable ("First 8-week") and, in concert with the Academic term name, fully communicate when a Course section is offered ("Fall 2018 First 8-week").
Instructional begin date InstrBeginDate The beginning of instruction for Course sections in the Session.
Instructional end date InstrEndDate The end of instruction for Course sections in the Sessions.
Inclusive access bill after date InclusiveAccessBillAfterDate The date after which inclusive access course materials can be billed to the institution.

Course offering

An institutional offering of organized subject matter and learner activities chosen by learners in an academic term.

The Course offering file should be created in a comma-separated (CSV) format with the naming convention course_offering_<date>.csv , where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Course offering ID SisIntId The unique primary key used by your SIS to internally identify a Course Offering. This ID may or may not differ from the external id.
SIS External Course offering ID SisExtId The unique, global id for the Course Offering generated for Canvas and other learning systems and tools.
SIS Internal Campus ID* CampusId The foreign key to the Campus where the Course offering is delivered.
SIS Internal Session ID* SessionId The foreign key to the Session to which the Course offering belongs.
Course subject CourseSubj

The alphabetic abbreviation of the academic department or discipline offering the course. It is one part of the total course Id number. E.g., "BIO" in "BIO 101."
Course number CourseNo

The official reference number portion of a course id. This number normally designates the level of the course as well as the level of the individual expected to enroll in the course. E.g. "101" in "BIO 101."
Course title Title

The title of the Course Offering.
Course description Description

The Course Offering description.
Academic career* Career

A code representing a grouping of academic programs for statistical purposes. An academic career exists for each school/college by the program level offered.

Option set: RefCareer
Course status Status

The status of the Course Offering at the institution.

Option set: RefOperationalStatus
Course credits AvailableCredits

Measured in Carnegie units, the amount of credit available to a student who successfully meets the objectives of the Course Offering.

Course section

An administrative and instructional sub-organization of a Course offering. Instructors may be assigned to one or more sections of a particular Course offering.

The Course section file should be created in a comma-separated (CSV) format with the naming convention course_section_<date>.csv , where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Course section ID SisIntId The unique primary key used by your SIS to internally identify a Course Section. This ID may or may not differ from the external id.
SIS External Course section ID SisExtId The globally unique, SIS-generated ID used by Canvas and other learning tools to identify a particular section.
SIS Internal Course offering ID CourseId The unique, SIS-generated ID for the Course Offering of which the section is an instance.
SIS Internal Academic organization ID* AcademicOrgId The foreign key to the Academic organization responsible for delivering this Course section.
Course Section number SectionNumber

The unique number of a particular Course Section. E.g., "1", "015".
Class number* ClassNumber A unique number for the Course section. This may or may not be the same as the internal SIS Course section identifier.
Section type* SectionType

The type of instruction delivered in this section. Institutions may refer to a section type as a "component" of a Course offering.

Option set: RefCourseSectionType
Delivery mode DeliveryMode

The delivery mode for the instruction given in this Course Section.

Option set: RefCourseSectionDeliveryMode
Max enrollment MaxEnrollment

Maximum number of students who can be enrolled in the Course Section.

Course grade*

The grade a person receives for a course as part of their Section enrollment.

While this field will likely be updated by Canvas during a Session, the SIS ultimately owns final, end-of-term grades. We expect this file will be blank until those grades are available.

The Course grade file should be created in a comma-separated (CSV) format with the naming convention course_grade_<date>.csv, where is the date on which the file was generated, and the following headers:

Data element Header Description
SIS Internal Course grade ID SisIntId The internal primary key used by the SIS to define a Course grade.
SIS External Course grade ID SisExtId The unique, global ID for a Course grade generated in your institutional SIS for use by external tools such as Canvas, LTI applications, etc.
SIS Internal Course section id CourseSectionId The foreign key to the Course section to which the Course grade belongs.
SIS Internal Person ID PersonId The foreign key to the Person to whom this Course grade belongs.
Score Score The absolute score for the Course grade. Often, this will be a numerical score ranging from 0-100 (but need not be). The intent is to capture a raw version of a course grade distinct from that reported in formal records.
Published score PublishedScore The published score for the Course grade. Often, this will be the letter version of the score. The intent is to capture the formal, published score on record (often, but not only, used by transcripts).
