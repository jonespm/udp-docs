Manifest file

Batch data integrations


Manifest files accompany every batch dataset pushed to the UDP for ingestion. They describe the contents of the full data dump, enabling the UDP to validate the completeness and integrity of the dataset beore ingestion. The schema for a manifest file is described below. 

  # The version of the manifest file used
  # in this dataset.
  manifest_version: v1

  # The name of the system whose data is ingested,
  # where values may include 'peoplesoft', 'banner',
  # 'canvas-data', etc. depending on the application.
  source: canvas-data

  # The UCDM version number this dataset's format
  # corresponds to.
  data_schema: 1.0

  ## The ISO 8601 datetime stamp describing when
  ## the dataset was produced.
  datetime: 2017-11-19T21:11:31Z #

  ## A UUID for the dataset
  dump_id: b4f8eec7-7adc-47a1-83a4-238f1032da00

  ## An array of the files included in the dataset.
  ## For each file, a checksum is provided.
  files:
    - name: assignment_dim_0.gz # name of file
      checksum: 291305b872ff5ae8e47dbe8d97ec23b2 # md5 checksum of file
    - name: assignment_dim_1.gz
      checksum: 163f2d7d097518435ccc61936e60cd49
    ...
